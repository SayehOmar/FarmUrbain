# Task: Fix Leaflet runtime error in Next.js project

## Context
The project throws this error:
TypeError: Cannot read properties of undefined (reading 'appendChild')

It occurs because Leaflet is trying to access the DOM during server-side rendering (SSR) in Next.js.

## Requirements
1. Refactor the code so Leaflet is only initialized on the client side.
2. Prevent SSR for the map component using `next/dynamic`.
3. Move map initialization into a `useEffect` hook.
4. Ensure the component is marked `"use client"`.
5. Fix any custom icon or map element initialization that depends on `document` or `window`.

## Implementation Steps
- Create a new component `/app/components/MapView.tsx`
- Inside it:
  - Add `"use client"`
  - Use `useEffect` to initialize the map
  - Make sure the container has an `id="map"`
  - Add a default OpenStreetMap tile layer
  - Center map on coordinates `[36.8065, 10.1815]` with zoom `12`
- In `/app/page.tsx` or `/app/urban-farm/page.tsx`:
  - Import the component dynamically:
    ```tsx
    const MapView = dynamic(() => import("../components/MapView"), { ssr: false });
    ```
  - Render it inside a container:
    ```tsx
    <main className="flex min-h-screen items-center justify-center">
      <div className="w-[90%] h-[80vh]">
        <MapView />
      </div>
    </main>
    ```
- Make sure all Leaflet markers and icons are defined after the map is initialized.

## Output
A working, client-side Leaflet map component in Next.js without SSR errors.
The runtime TypeError should be completely resolved.
